# Contributing

Thank you for your interest in contributing to this Terraform module!

## Prerequisites

- [Terraform](https://developer.hashicorp.com/terraform/install) >= 1.5.0
- [pre-commit](https://pre-commit.com/#install)
- [terraform-docs](https://terraform-docs.io/user-guide/installation/)
- [tflint](https://github.com/terraform-linters/tflint)
- [trivy](https://aquasecurity.github.io/trivy/)

## Getting Started

1. Fork the repository
2. Clone your fork
3. Install pre-commit hooks:
   ```bash
   pre-commit install
   ```
4. Create a feature branch from `main`:
   ```bash
   git checkout -b feature/my-feature
   ```

## Development Workflow

### Code Style
- Run `terraform fmt -recursive` before committing
- All variables and outputs must have descriptions
- Use `snake_case` for all Terraform identifiers

### Testing
Run unit tests (no Azure credentials needed):
```bash
terraform test -filter=tests/unit
```

Run integration tests (requires Azure credentials):
```bash
terraform test -filter=tests/integration
```

### Documentation
Documentation is auto-generated by [terraform-docs](https://terraform-docs.io/). Update variable/output descriptions in `.tf` files and run:
```bash
terraform-docs .
```

### Commit Messages
Use [Conventional Commits](https://www.conventionalcommits.org/):
- `feat:` new feature
- `fix:` bug fix
- `docs:` documentation changes
- `refactor:` code refactoring
- `test:` adding or updating tests
- `chore:` maintenance tasks

### Pull Requests
- Target the `main` branch
- Fill out the PR template completely
- Ensure CI checks pass (fmt, validate, lint, security scan, tests)
- Update `CHANGELOG.md` under the `[Unreleased]` section

## Reporting Issues

Please use the [GitHub issue tracker](../../issues) with the appropriate template.
